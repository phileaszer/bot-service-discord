Installation

Installer Sentinel proprement.

Le bot doit etre ajoute comme bot Discord, pas seulement comme integration de commandes.

Inviter Sentinel Aide installation

Etapes

Demarrage recommande.

Suis ces etapes dans l'ordre pour eviter les problemes de permissions et de roles.

  1. Inviter SentinelUtilise le lien officiel avec les scopes bot et applications.commands.
  2. Verifier l'integrationDans les integrations Discord, Sentinel doit avoir le badge Bot.
  3. Placer le role SentinelLe role du bot doit etre au-dessus du role de service et des roles a moderer.
  4. Choisir la langueUtilise /config-langue ou le dashboard pour choisir FR ou EN.
  5. Configurer le serviceDefinis le role de service, le salon de logs et les roles autorises.
  6. Publier le panneauEnvoie !service-panel ou utilise le dashboard.

Panneau d'aide Discord

Le mode d'emploi integre au bot.

Sur Discord, la commande /aide affiche le tutoriel complet de Sentinel. Depuis le site, tu peux suivre ce deroule pour installer, configurer et utiliser le bot sans connaitre Discord.

01

Installer

Invite Sentinel sur ton serveur, puis lance /config-langue pour choisir Francais ou English.

02

Configurer

Definis le role de service avec /config-role, le salon de logs avec /config-logs, puis les roles autorises avec /config-permissions.

03

Utiliser

Publie le panneau avec !service-panel. Les membres pourront prendre service, voir leurs heures et suivre les agents actifs.

04

Verifier

Utilise /config-voir pour relire la configuration et /reset-heures pour remettre a zero une personne, meme par ID.

Depannage / Troubleshooting

Si Discord bloque l'invitation.

Ces points reglent la plupart des soucis quand Sentinel ne rejoint pas le serveur, n'apparait pas comme bot, ou quand le dashboard ne semble pas agir sur le bon serveur.

01

Verifier les droits Discord

La personne qui invite Sentinel doit avoir la permission Gerer le serveur ou Administrateur sur le serveur choisi.

EN: the installer needs Manage Server or Administrator permission on the selected guild.

02

Choisir le bon serveur

Si Discord ouvre directement un serveur qui n'est pas le bon, ferme la fenetre et relance le lien officiel depuis cette page sans lien personnalise.

EN: if Discord locks the wrong guild, restart from the official invite button on this page.

03

Confirmer que le bot est bien membre

Dans les integrations Discord, Sentinel doit apparaitre avec le badge Bot. Si tu vois seulement Commandes, retire l'integration et reinvite-le.

EN: Sentinel must show as a Bot integration, not only as slash commands.

04

Tester apres l'installation

Une fois Sentinel arrive sur le serveur, utilise /ping puis /diagnostic pour verifier la base, les roles et les permissions.

EN: after inviting, run /ping and /diagnostic to confirm that everything is ready.

Important

Le lien officiel contient deux autorisations.

Sentinel a besoin de bot et applications.commands. Sans les deux, les commandes peuvent apparaitre sans que le bot soit vraiment present sur le serveur.
