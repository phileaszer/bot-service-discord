Installation

Installer Sentinel proprement.

Sentinel doit être ajouté comme bot Discord, pas seulement comme intégration de commandes.

Étapes

Démarrage recommandé.

Suis ces étapes dans l’ordre pour éviter les problèmes de permissions et de rôles.

  1. Inviter SentinelUtilise le lien officiel avec les scopes bot et applications.commands.
  2. Vérifier l’intégrationDans les intégrations Discord, Sentinel doit afficher le badge Bot.
  3. Placer le rôle SentinelLe rôle du bot doit être au-dessus du rôle de service et des rôles à modérer.
  4. Choisir la langueUtilise /config-langue ou le dashboard pour choisir FR ou EN.
  5. Configurer le serviceDéfinis le rôle de service, le salon de logs et les rôles autorisés.
  6. Publier le panneauEnvoie !service-panel ou utilise le dashboard.

Dépannage / Troubleshooting

Si Discord bloque l'invitation.

Ces points règlent la plupart des soucis quand Sentinel ne rejoint pas le serveur, n’apparaît pas comme bot ou quand le dashboard ne semble pas agir sur le bon serveur.

01

Vérifier les droits Discord

La personne qui invite Sentinel doit avoir la permission Gérer le serveur ou Administrateur sur le serveur choisi.

EN: the installer needs Manage Server or Administrator permission on the selected guild.

02

Choisir le bon serveur

Si Discord ouvre directement un serveur qui n'est pas le bon, ferme la fenêtre et relance le lien officiel depuis cette page, sans lien personnalisé.

EN: if Discord locks the wrong guild, restart from the official invite button on this page.

03

Confirmer que le bot est bien membre

Dans les intégrations Discord, Sentinel doit apparaître avec le badge Bot. Si tu vois seulement Commandes, retire l’intégration et réinvite-le.

EN: Sentinel must show as a Bot integration, not only as slash commands.

04

Tester après l’installation

Une fois Sentinel arrivé sur le serveur, utilise /ping puis /diagnostic pour vérifier que les rôles et les permissions sont prêts.

EN: after inviting, run /ping and /diagnostic to confirm that roles and permissions are ready.

Important

Le lien officiel contient deux autorisations.

Sentinel a besoin de bot et applications.commands. Sans les deux, les commandes peuvent apparaître sans que le bot soit vraiment présent sur le serveur.
